1Verdikt: geht das?

Ja — und es ist einfacher, als es klingt.

Das Questsystem von Two Worlds 1 ist ein generischer, datengetriebener Loader. Der kompilierte Missions-Skript PQuests.eco ruft LoadQuests1("Scripts\Quests\TwoWorldsQuests.qtx") auf, und der Parser im SDK-Quelltext (PQuestLoader.ech) verzweigt nur über die Textmarken QUEST / GIVER / FC / AOQ / ACTION / REWARD / NPC / END. Es gibt keine fest verdrahtete Logik pro Quest. Ein neuer QUEST-Block ist mechanisch eine neue Quest.

✓
Kernaussage. Eine neue Quest zu bauen bedeutet, zwei Textdateien zu erzeugen und sie als Mod ins Spiel zu legen. Kein Reverse Engineering, kein Editor zwingend nötig. Das Werkzeug in diesem Ordner nimmt dir beides ab.

Was statisch bewiesen ist:

  • Die drei Formate (.qtx, .lan, .wd) sind vollständig gelesen und geschrieben — der .lan-Writer reproduziert die 3,3-MB-Originaldatei byte-genau, der .qtx-Emitter alle 500 Original-Quests byte-genau.
  • Die Verknüpfung Quest → Text läuft rein über Namenskonvention (translateQ_<n>), aus dem SDK-Quelltext bestätigt.
  • Neuer Text kann additiv in einer neuen Language\*.lan ausgeliefert werden — der Master bleibt unangetastet.
  • Eine neue Quest braucht keine Karten-/Marker-Arbeit, wenn sie einen vorhandenen Quest-NPC als Geber wiederverwendet.

Was noch offen ist (der eine Test):

  • Dass eine Mod-Datei die Basis wirklich überschreibt — du weißt aus deiner PAR-Mod, dass die neueste Datei gewinnt; das bestätigen wir mit dem Smoke-Test.
  • Dass das Spiel die neue Quest-ID zur Laufzeit akzeptiert (kein Absturz).

2Wie eine Quest aufgebaut ist

Ein QUEST-Block bündelt Geber, Ziele, Verkettung, Aktionen und Belohnungen.

So sieht eine echte, kleine Original-Quest aus (aus Net_M_20.qtx):

QUEST Q_2007 1 10 (null) 0 True
  GIVER PASSIVE NPC_2007 AUTO_CLOSE_ON_SOLVE NONE
  FC BRING_OBJECT QITEM_096 1        # Ziel: bring 1× Gegenstand
  REWARD ITM SOLVE -1 QITEM_096       # nimmt den Gegenstand wieder
  REWARD EXP SOLVE SMALL
  REWARD GLD SOLVE SMALL
  AOQ CLOSE SOLVE Q_2006             # schließt die Eltern-Quest
END

3Die Pipeline

Reality Pump nutzte WhizzEdit. Wir gehen direkt auf die kompilierten Formate — kürzer und ohne Altlasten.

4Formatreferenz

4.1  .qtx — die Quest-Definitionen verifiziert

Reiner ASCII-Text, LF-Zeilenenden, (null) = leer. Drei Satzarten auf oberster Ebene, Unterzeilen exakt zwei Leerzeichen eingerückt, Blöcke mit END in Spalte 0.

UnterzeileBedeutungBeispiel
GIVERGeber-NPC + VerhaltenGIVER PASSIVE NPC_145 BACK_TO_GIVER_MAP_SIGN NONE
FCAbschlussbedingung (Ziel)FC KILL NPC_231 · FC BRING_GOLD 500 · FC GO 23 D1 0
AOQQuest-VerkettungAOQ TAKE TAKE Q_380
ACTIONWelt-EventACTION NPC_CREATE TAKE NPC_231
REWARDBelohnungREWARD GLD CLOSE MEDIUM · REWARD ITM SOLVE 1 QITEM_096

Der Kopf ist QUEST <id> <EnableLevel> <Group> <Guild> <MinRep> <InsLog>. Die vollständige Arity-/Enum-Tabelle steckt in tw1_qtx.py und validiert jede erzeugte Zeile.

4.2  .lan — die Texte verifiziert

Binär, little-endian. Der Schlüssel trägt das Präfix translate. Eine minimale reine Text-Datei ist Kopf + N Übersetzungen + zwei Null-Zähler:

"LAN\0" | u32 version=3
u32 N   ; N × { u32 keyLen, ASCII-Key, u32 valLen(Zeichen), UTF-16LE-Wert }
u32 0   ; Aliasse
u32 0   ; Dialogbäume

Die Schlüssel pro Quest Q_<n>:

SchlüsselInhalt
translateQ_<n>Quest-Titel
translateQ_<n>_QTDJournaltext beim Annehmen
translateQ_<n>_QSDJournaltext beim Lösen (optional)
translateQ_<n>_QCDJournaltext beim Abschließen
translateGROUP_<n>Name einer Quest-Gruppe (eigene Überschrift im Tagebuch)
⚠️
Überschrieben wird auf Datei-Ebene, nicht auf Schlüssel-Ebene. Das Spiel lädt die neueste Datei unter einem bestimmten Pfad. Eine neue Datei (Language\ZZ_MeinMod.lan) wird nie geladen — danach sucht niemand. Quest-Text heißt also: Master lesen, eigene Schlüssel hineinmischen, die ganze Datei unter dem Originalpfad Language\TwoWorldsQuests.lan wieder ausliefern. (Im ersten Test genau daran gescheitert.)

4.3  .wd — das Archiv verifiziert

Alle Spieldateien liegen in .wd-Archiven (zlib-komprimiert). Aufbau:

tw1_wd.py liest und schreibt das — round-trip byte-genau gegen deine eigene Yamalin.wd getestet, inkl. Erhalt der Flags und Extra-Felder.

5Das QuestForge-Werkzeug

Eine kleine JSON-Datei rein, eine fertige Mod-.wd raus.

Beispiel-Spec (example_quest.json):

{
  "mod_name": "MyFirstQuest",
  "quests": [{
    "id": "auto",                         # nimmt die nächste freie ID (381–399)
    "title": "Der Prüfstein",
    "journal_take": "Sammle 500 Goldstücke.",
    "journal_close": "Du hast das Gold gebracht.",
    "giver": { "npc": 145, "type": "PASSIVE",
               "type_ex": "BACK_TO_GIVER_MAP_SIGN", "remove_time": "NONE" },
    "objectives": [{ "fc": "BRING_GOLD", "args": [500] }],
    "rewards": [
      { "type": "GLD", "time": "CLOSE", "args": ["MEDIUM"] },
      { "type": "EXP", "time": "SOLVE", "args": ["MEDIUM"] }
    ]
  }]
}

Bauen und ausliefern:

# nur bauen -> out\MyFirstQuest.wd
python questforge.py example_quest.json

# bauen + in den Mods-Ordner kopieren + in der Registry aktivieren
python questforge.py example_quest.json --deploy --register

# eine Starter-Spec ausgeben
python questforge.py --example
🛡️
Das Werkzeug validiert jede Zeile (richtige Anzahl Tokens, gültige Enum-Werte), hält die Quest-ID im sicheren Bereich, prüft dass der Geber-NPC existiert, erzwingt LF, und liest die fertige .wd zur Kontrolle mit dem Spiel-eigenen Leser wieder ein — bevor sie geschrieben wird.

6Die harten Regeln

Vier Dinge, die man falsch machen kann — das Werkzeug hält sie automatisch ein.

RegelWarum
Nur Quest-IDs 381–399Der Singleplayer-Loader kappt bei 400. Die IDs 700+ gehören dem Multiplayer — eine Quest dort wäre im SP unsichtbar.
Nur LF, niemals CRLFDer Parser trennt an Leerzeichen; ein \r verseucht das letzte Token jeder Zeile (True\r) und bricht die ganze Datei.
Von Update16 ausgehenDie Patches 1.1–1.6 überschreiben die Quest-Datei. Von der alten 1.0-Kopie auszugehen würde alle Patch-Fixes verwerfen.
Bestehenden Geber-NPC nutzenDann keine Marker-/.lnd-Arbeit im Editor nötig — der NPC hat seinen MARKER_QUEST_START schon.
⚠️
Geber-Quests brauchen einen Dialogbaum. Alle 377 Original-Quests mit Geber haben einen DQ_<n>-Dialog. Für die erste Quest umgeht man das am besten mit einer giverlosen Quest, die über AOQ von einer bestehenden Quest ausgelöst wird — oder man akzeptiert, dass der Geber die Quest ohne gesprochenen Dialog anbietet (unbewiesen). Der Dialogbaum-Autor ist der nächste Ausbauschritt des Werkzeugs.

7Ausliefern: Mods + Registry

Die neueste Datei gewinnt — genau wie bei deiner PAR-Mod.

Aktiviert wird über einen Registry-Wert (macht --register automatisch):

HKCU\SOFTWARE\Reality Pump\TwoWorlds\Mods
    "MeinMod.wd" = 1   (DWORD; 1 = an, 0 = aus)

8Der In-Game-Test wartet auf dich

Ein 1-Minuten-Test, der alle offenen Fragen auf einmal klärt.

Der Smoke-Test-Mod ist bereits gebaut, in den Mods-Ordner kopiert und in der Registry aktiviert (QuestTest.wd = 1). Er tut zwei Dinge:

  • Signal A — er stellt jedem sichtbaren Quest-Titel ein [MOD] voran (über eine neue .lan).
  • Signal B — er hängt eine neue Quest Q_385 an die .qtx an.
So testest du:
  1. Two Worlds starten (normal, Singleplayer).
  2. Einen Spielstand laden.
  3. Questlog öffnen (J bzw. übers Menü).
  4. Steht [MOD] vor deinen Quest-Titeln?

→ Ja: Präzedenz + .lan-Override + Laufzeitakzeptanz sind bewiesen. Das Spiel lädt Mod-Quests. Grünes Licht für echte Quests.
→ Nein: Sag mir Bescheid — dann liegt es an der Ladereihenfolge, und ich habe eine Fallback-Variante (loser Ordner statt .wd) parat.

↩️
Rückgängig: in der Registry QuestTest.wd = 0 setzen (oder die QuestTest.wd aus dem Mods-Ordner löschen). Deine Saves bleiben unberührt — nur die Anzeige-Titel ändern sich.

10Vertonung

Aufgeschoben, bis die erste Quest läuft — aber der Plan steht.

Der Ton liegt in XACT\win\ (nicht in den .wd-Archiven): eine 935-MB-Wave-Bank UnitTalk.xwb plus Sounds.xsb. Neue Sprachzeilen heißt diese Bank neu bauen — deutlich mehr Arbeit als die Quest selbst. Aber: Dialogzeilen ohne Sprachdatei sind im Format vorgesehen (cue=""), also funktionieren neue Quests zunächst mit Text/Untertiteln.

Plan: neue NPC-Stimmen über ElevenLabs (günstige, passende Stimmen), für den Helden eine ähnliche Stimme suchen. Das kommt, sobald der In-Game-Test grün ist — vorher lohnt kein Abo.
